[ 게임 개요 ]
테트리스 + 젠가를 결합한 2D 퍼즐 게임.
4x10 그리드에서 블럭을 선택해 빼고, 위에 다시 올려 타워를 높이는 방식.

================================================================
[ 구현 완료 항목 ]
================================================================

■ 핵심 게임 로직
  - 4x10 그리드 초기화 (각 블럭 숫자값 2~6 랜덤 배정)
  - 마우스 클릭으로 블럭 선택 (최대 4개, 인접+연결된 블럭만)
  - 선택 블럭 들어올리기 (LiftBlocks)
  - 들어올린 블럭을 타워 위에 배치 (TryPlaceBlocks)
  - 들기 취소 기능

■ 선택 규칙
  - 상하좌우 인접한 블럭만 연속 선택 가능 (대각선 불가)
  - 선택 해제 시 나머지가 연결 상태 유지되는지 BFS로 검증
  - 최대 4개 제한

■ 배치 규칙
  - 이미 점유된 칸에는 배치 불가
  - 배치 가능 영역: (0,0) ~ (3,14) (15행까지 쌓기 가능)
  - 기존 타워와 인접한 위치에만 배치 가능
  - [신규] 중력 적용: 아래에 지지 블럭이 없으면 배치 불가

■ 물리 / 시뮬레이션
  - TowerRoot 오브젝트 단일 Rigidbody로 타워 전체 물리 적용
  - 블럭 제거 후 고립 블럭 자동 감지 및 분리 (CheckForDetachment)
  - 분리된 블럭은 개별 Rigidbody로 낙하

■ 점수 시스템
  - 배치 1회 = +1점 (화면 좌측 상단 표시)

■ 게임 오버
  - 감지 조건: 미구현 (게임오버 화면은 존재하나 트리거 조건 없음)
  - 게임오버 화면: "GAME OVER" + 점수 + 클릭 시 재시작

■ 카메라
  - 직교(Orthographic) 카메라
  - 화면 비율에 맞게 게임 전체가 보이도록 자동 조절 (FitCamera)
  - 마우스 휠로 상하 스크롤 가능

■ UI / 시각
  - 블럭 숫자별 색상 구분
  - 선택된 블럭 하이라이트 (글로우 효과)
  - 경계선(빨간 선) 좌우 표시
  - 점수 레이블 (TextMeshPro)

[ 미구현 / 미완성 항목 ]
================================================================

□ 게임오버 트리거 조건
    현재 GameOverScreen은 존재하지만 어떤 조건에서 호출할지 정의 없음
    (예: 타워가 경계선 밖으로 쓰러질 때, 특정 높이 초과 시 등)

□ 블럭 배치 시각 피드백
    현재 배치 불가 위치에 cursor를 가져가도 아무 시각적 표시 없음
    (배치 가능/불가 색상 미리보기 없음)

□ 사운드 / 이펙트
    효과음, BGM, 배치 이펙트 등 전혀 없음

□ 난이도 / 레벨 시스템
    스테이지 진행, 난이도 변화 없음

□ 블럭 배치 미리보기 (Ghost)
    들고 있는 블럭이 놓일 위치를 반투명으로 미리 보여주는 기능 없음

□ 타워 안정성 판정
    물리 시뮬레이션은 있으나, 실제 '타워가 무너지는' 연출/판정 미구현

[ 알려진 버그 및 원인 분석 ]
================================================================

■ 버그 1: 특정 위치에 블럭을 놓을 수 없음
  원인:
    배치 조건이 "기존 타워 블럭과 상하좌우 1칸 인접"을 요구함.
    인접 조건을 만족하지 못하면 배치가 조용히 실패함 (아무 반응 없음).

  오해 포인트:
    "4면이 다 맞닿아야 하는 건가?" → 아님.
    4방향 중 단 1방향만 인접해도 배치 허용.
    단, 블럭 바로 아래에 받쳐주는 블럭이 없으면
    [신규 중력 체크]에 의해 배치 불가 (아래 버그 2 수정 내용 참조).

  현재 상태:
    배치 불가일 때 시각적 피드백이 없어 유저가 이유를 모름.
    → 배치 불가 위치 표시 기능 추가 권장.

■ 버그 2: 중력 미적용 - 아래 블럭 없이 위 블럭에 붙어서 배치 가능 [수정 완료]
  원인:
    TowerRoot가 단일 Rigidbody라서 블럭이 개별적으로 떨어지지 않음.
    배치 시 그리드 좌표에 즉시 고정되고, 아래 지지 블럭 유무를 확인 안 했음.
    그 결과: 위쪽 블럭에만 옆으로 인접해도 공중에 배치 가능한 상태였음.

  수정 내용 (TryPlaceBlocks 내 중력 체크 추가):
    배치하려는 블럭 그룹 중, 그룹 내부에서 바로 아래 블럭이 없는 블럭은
    반드시 기존 타워 블럭 위 또는 바닥(row 0)에 놓여야 배치 허용.
    → 공중 부양 배치 차단.
